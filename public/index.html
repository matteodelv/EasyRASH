    EasyRASH Project Info

    EasyRASH is a simple web application that lets you easily manage conferences and academic events by focusing on the Peer Review process for articles written in RASH. Click on the Help button on top to discover all the features of EasyRASH, its usage and icon meanings.

    EasyRASH Help

    Here you can find some information about how to use EasyRASH and how to review an article or manage an event, icon meanings and a small description of the interface.

    Interface

    Navigation Bar

    Located on top just below the banner, it contains key elements to interact with events and papers. On the left side, a Mode Toggle button lets every reviewer to enter in Annotator Mode for adding annotations to a paper as a review. Furthermore, there are two hidden buttons to let every chair manage an event: in fact, they will only appear when the logged in user is one of the chairs of the currently selected conference. The first button shows the Management Panel for a conference, where every chair can edit its info, and add co-chairs or reviewers; the second button, instead, is used by every chair to assign reviewers to the currently open paper. The last button will let the chair make a final decision about the shown paper.

    On the right side, there are an Help button (which opens this modal) and an Info button that lets you see our pretty faces.

    Main Area

    It is the biggest part of the interface and it is used to show papers, for both reading and review, and panels to edit various kind of information. In some specific conditions, based on paper status and user's role, annotations will also be displayed. In addition, papers can be displayed both in RASH and Springer LNCS layout. A sidebar affixed on the right will help you scroll through the different sections and keep track of where you're at.

    Sidebar

    Located on the left, the sidebar is the main way to navigate between papers and conferences. On the top, some info about the currently logged in user are displayed, a Edit Profile button and a Log Out button. Below there can be found a Conferences Selector, where users can select specific events and get papers from that event, or they can create new conferences. The last section, divided in Authored By Me and Pending Reviews, shows a list of all the articles the logged in user is author of and all the papers that the user has yet to review.

    Note: This last section can be considered as a summary of the work related to the user, no matter which conference is or is not selected.

    Frequently Asked Questions

    How can I create a new conference?

    You can find a New Conference entry in the drop down button on the sidebar on the left. Clicking on that button will open a modal that lets you create a new conference.

    How do I review a paper?

    First of, enter the Annotator Mode by pressing the button on the top bar. Then select a portion of text you would like to annotate and click the button that pops up with the + sign. You will be prompted with a small text editor which lets you type your annotation. Rinse and repeat. Once you're happy with your annotations, click on the Review Annotations button on the top bar, check that everything is ok and choose whether you think the paper should be accepted and rejected, and confirm your choice.

    Why can't I see the other reviewers annotations?

    Annotations are displayed only in specific circumstances:

    • If the paper has been accepted by the chair.
    • If the user is chair of the event the paper belongs to.
    • If the user is a reviewer of that paper, and has already reviewed it, to prevent the other reviews to influence him.
    • The paper isn't pending, and the user is either the author of the paper or a potential reviewer (PC member).
